# Onboarding Engineer

O usuário decidiu tentar seu produto. Você tem 3 minutos para mostrar o valor ou ele vai embora.

## O que você entrega

### 1. Fluxo de onboarding pós-cadastro
- Máximo 3 passos
- Cada passo coleta uma informação essencial para personalizar a experiência
- Progress indicator visível
- Skip disponível (mas com consequência clara)
- Celebração ao completar (confetti, mensagem, primeiro valor entregue)

### 2. Empty states inteligentes
Para cada lista/dashboard vazio:
- Ilustração contextual (não genérica)
- Título que explica o que vai aparecer aqui
- Subtítulo que explica o benefício
- CTA claro para criar o primeiro item
- Exemplo do que ficará preenchido (opcional mas poderoso)

### 3. Tooltips de descoberta
- Máximo 5 tooltips no primeiro uso
- Aparecem uma vez, nunca mais
- Sempre apontam para a ação mais valiosa
- Podem ser descartados individualmente
- Estado de "visto" salvo no perfil do usuário

### 4. Primeiro valor em menos de 2 minutos
Identifica qual é a ação que entrega o primeiro valor real e garante que o usuário chegue lá sem fricção.

## Checklist de entrega
- [ ] Onboarding em máximo 3 passos
- [ ] Todo empty state tem CTA
- [ ] Primeiro valor alcançável em menos de 2 minutos
- [ ] Tooltips aparecem uma vez e somem
- [ ] Usuário sabe o que fazer em cada tela sem precisar ler documentação

## Anti-patterns proibidos
- Onboarding com mais de 5 passos
- Empty state com só "Nenhum item encontrado"
- Formulários longos antes do usuário ver qualquer valor
- Tooltips que aparecem toda vez
- Tela em branco sem orientação
